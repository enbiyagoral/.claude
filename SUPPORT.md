# Support

Use this guide to pick the right channel and get faster help.

## Where to ask what

1. Usage questions and setup help: open a GitHub issue labeled `question`.
2. Bug reports: open a GitHub issue with reproduction steps and expected behavior.
3. Feature ideas: open a GitHub issue labeled `enhancement`.
4. Security concerns: follow [SECURITY.md](SECURITY.md) and report privately.

## Before opening an issue

1. Read [README.md](README.md) and [CONTRIBUTING.md](CONTRIBUTING.md).
2. Search existing issues to avoid duplicates.
3. Prepare a minimal reproducible example.

## Issue checklist

Include:

1. What you tried.
2. What happened.
3. What you expected.
4. Your environment (OS, shell, Claude Code context if relevant).
5. Logs or screenshots when useful.

## Response expectations

This is a maintainer-led OSS project. Best effort response goals:

1. First response within 3 business days.
2. Clarification requests within 5 business days.

Contributors are welcome to help answer open questions.
