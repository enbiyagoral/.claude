# Security Policy

## Supported versions

This repository is a template project. Security fixes are provided on the latest `main` branch.

## Reporting a vulnerability

If you discover a security issue, do not open a public issue first.

Please report privately using GitHub Security Advisories:

1. Go to `Security` tab in this repository.
2. Click `Report a vulnerability`.
3. Provide reproduction steps, impact, and affected files.

If GitHub Security Advisories are unavailable, contact the maintainer privately on GitHub.

## What to include

Include as much detail as possible:

1. Vulnerability type and impact.
2. Exact reproduction steps.
3. Proof-of-concept or minimal example.
4. Suggested mitigation if known.

## Response targets

Maintainer target SLAs:

1. Initial acknowledgment within 24 hours.
2. Triage and severity assessment within 7 days.
3. Fix timeline communicated after triage.

## Disclosure policy

Please allow time for investigation and patching before public disclosure. After a fix is released, coordinated disclosure is welcome.
